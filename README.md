# 五竜陣のオンラインゲーム化について
## 実装案
- RecordManagerをNetworkBehaviourにして、盤面全体を共有するのではなく、新しい一手をどちらにも追加されるようにする。
- プレイヤーは先手か後手か固定し、 今どちらのターンかはNetworkBehaviourで共有し入力を制御する。
## 進め方
はじめ
- ローカルのスクリプトの方で描画処理など転用できるものは転用して、オンライン化においてどうしても共有や変更しないといけないものは別途準備する。
- まず、オンライン化するに当たって盤面の二次元配列ではなくどの手を打ったのかの履歴を同期対象にする。そして、その手の履歴をもとに各自ローカルで全体の盤面を再生する。この盤面をもとにどの手を打ったほうがいいのかを判定し、その処理を履歴化し同期対象である履歴を更新する。
